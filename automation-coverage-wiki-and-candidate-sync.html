Automation Coverage — Wiki Registry & Candidate Sync

Two PRs, one complete picture. #1986 moved the registry out of the repo and into the GitHub Wiki — so it regenerates automatically on every merge, with no PR conflicts. #1989 added the inbound flow: pull newly-flagged Jira candidates into the backlog with a single command.

#1986 Registry → GitHub Wiki (auto-regenerated)

Before this PR, AUTOMATION_COVERAGE.md was committed to mobile_dev. Every PR that touched a test file had to regenerate it and commit it — causing conflicts whenever two PRs were open at once.

What changed

  • AUTOMATION_COVERAGE.md is now gitignored — never committed to the repo.
  • After every merge to mobile_dev, the sync-test-registries CI workflow regenerates the file from code and pushes it to the GitHub Wiki.
  • Same for DISABLED_TESTS.md — both registries now live in the wiki.
  • PRs no longer need a "regenerate and commit" step — just merge and the wiki updates within ~30s.

#1989 Pull New Candidates from Jira

The registry shows what's already automated. This PR adds the other half: pulling in Jira tickets that QA flagged as mobile-sanity candidates but haven't been automated yet — so they appear in the backlog and can't silently stay off the radar.

The command

/automation-coverage sync from jira

That's it. Claude exports the candidates from Jira, reconciles them into regression-suite.csv, and the wiki refreshes after merge. No token or MCP required — just a CSV export from the Jira filter UI.

What gets pulled in

Only tickets that are Testing Level = Sanity and Platform type = Mobile / Candidate — scoped to the domains we track (TeamBanking + page domains). Out-of-scope domains (Securities, Chat, Open Banking…) are skipped. Unrecognised domains go to a triage file for a human to review.

How the flow works end-to-end

1
QA flags a TTM Test Case in Jira: Platform type = Mobile / Candidate, Testing Level = Sanity.
2
Run /automation-coverage sync from jira in Claude Code — exports the saved filter and reconciles into regression-suite.csv.
3
Commit the updated CSV via a normal PR. The wiki AUTOMATION_COVERAGE.md refreshes automatically ~30s after merge — no extra step.
4
The develop-uat CI run summary echoes a short reminder to repeat the sync whenever new candidates are flagged.

Also in #1989

  • Out-of-scope domain highlight — if a domain is removed from TeamBanking, its CSV rows appear in a clearly-labelled ⚠️ Not in TeamBanking scope section instead of silently rendering as normal.
  • 27 unit tests covering the CSV ingestion, domain classification, and reconciliation logic.
  • 22 Dashboard backlog rows seeded — the Dashboard section went from 10/10 to 10/32 automated, reflecting real outstanding work.
