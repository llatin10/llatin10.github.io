3
Make Card Marketing Component Configurable
Both Feature Enhancement

Objective: Make marketing components in the cards-new-lobby screen configurable via remote configuration (Firebase key: cards_lobby_promotions) instead of hard-coded values, enabling updates without requiring a new app version.

Configuration:

  • Each card contains: image, link (deep link or external URL), and text (header and body managed via Lokalise)
  • Cards are displayed conditionally based on:
    • planType: "ZERO", "ONE", "ONE_PLUS", "ONE_FOUNDER", "ONE_PLUS_FOUNDER"
    • connectedToOpenBanking: boolean (true/false)

Card Display Logic:

  • ZERO plan + Open Banking: 3 cards (zero_exchange_fees, cashback, isracard_benefits)
  • ZERO plan + No Open Banking: 4 cards (includes connect_open_banking)
  • ONE plan + Open Banking: 2 cards (cashback, isracard_benefits)
  • ONE plan + No Open Banking: 3 cards (includes connect_open_banking)

Analytics: Includes impression and click events per card with promotion_id parameter.

Business Value: Enables switching providers (e.g., eSIM from Airalo to new company, travel insurance from Isracard/AIG to Harel) without app updates.
