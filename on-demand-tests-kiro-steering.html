Kiro Steering File — OnDemand specific tests

Use this guide when working in Kiro to generate the exact string to paste into the GitHub Actions workflow field:

Run specific test class pattern(s), comma-separated. Takes precedence over testTag.

This is the Kiro equivalent of the Cursor /on-demand-specific-test-run command and the Firebender prompt.

How to use in Kiro

In the Kiro chat, type #on-demand-specific-test-run followed by your test names:

#on-demand-specific-test-run
Mobility - mobility request - Joint Account
downgrade from existing plan
Dashboard - balances list - open banking is not connected - navigation

Kiro will automatically:
1. Look up each test name in the mobile-tests index
2. Resolve it to the correct Kotlin class name
3. Output a ready-to-paste copy-paste line + verification table

What Kiro does under the hood

Kiro runs the extract_test_index.py script against the mobile-tests repo:

python3 ~/.kiro/imports\ from\ cursor/version-prep/scripts/extract_test_index.py \
  --repo /Users/llatin/mobile-tests \
  --find "partial test description"

The script searches src/test/kotlin for:
- Backtick test titles: fun `...`(
- Parameterized display templates: @ParameterizedTest(name = "...")

Output format

1. Copy-paste line (paste directly into GitHub Actions):

*MobilityFullFlowJointAccountTest,*DowngradeFromExistingPlanTest,*UpgradeFromExistingPlanTest

2. Verification table:

# Description Test class
1 Mobility - mobility request - Joint Account MobilityFullFlowJointAccountTest
2 downgrade from existing plan DowngradeFromExistingPlanTest
3 upgrade from existing plan UpgradeFromExistingPlanTest

Rules

  • Each entry uses a leading * plus the Kotlin class name (no package, no quotes)
  • Dedupe classes — if several tests map to the same class, one entry is enough
  • If no match is found, Kiro will say so in the table and omit it from the copy-paste line
  • The copy-paste line is always a single line, no line breaks

Steering file location

~/.kiro/steering/on-demand-specific-test-run.md

This is a global steering file — available in all your Kiro projects.
