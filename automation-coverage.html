Automation Coverage by Domain — Code-Derived & Hands-Free

The Automation by Domain Progress Confluence page was maintained by hand, so its ✅ marks drifted from reality. PRs #1962 and #1964 turn it into a checked-in, code-derived registry (AUTOMATION_COVERAGE.md) that the team never has to maintain — the same idea #1955/#1960 did for disabled tests, now with a hands-free twist. (Both PRs open; this is the shipping plan.)

✨ Key Highlights

🪄 Hands-free: a bot regenerates the registry and commits it back to your PR — you just push your code.

Derived from code: Automated / Android / iOS marks come from @TmsLink + enabled state, so they can't drift.

Auto-include: any in-scope test is tracked the moment it gets a @TmsLink — impossible to silently miss.

Shared scope: the same TeamBanking tag group as the disabled-tests registry — the two can't disagree.

CI gate fails a PR if the registry drifts — but the bot fixes it first, so you rarely see red.

/automation-coverage (#1964): one command to sync the Confluence page and curate summaries.

What "Automated" means (read from the code)

A test is Automated ✅ if a @TmsLink("OZ-…") for it exists in the repo — even if it's disabled, broken, or commented out. Android ✅ / iOS ✅ means it's actively running on that platform (enabled, not platform-disabled). So a disabled-but-automated test reads ✅ | (blank) | (blank).

@TmsLink("OZ-74848")        // -> Automated ✅ (and Android/iOS if it runs there)
@Test
fun `Mandates - Create New Mandate`() { ... }

🪄 The hands-free part

The same sync-test-registries.yml workflow that keeps disabled-tests honest also keeps this one honest: on every PR it regenerates AUTOMATION_COVERAGE.md and, if it's stale, commits the fix back to your branch as github-actions[bot]. No script to run, no hook to install. Repo-scoped GITHUB_TOKEN (no long-lived secret); the --check gate stays as the backstop.

Where the numbers come from

The ✅ are computed from code; only the curated "must-run" backlog (tests with no code yet), the human summaries, and the out-of-scope page domains (Payments / Movements / Cards / Limits) live in one hand-edited file, scripts/automation-coverage/regression-suite.csv.

The 39 "auto-included" = a migration worklist, not page rows

39 in-scope tests are automated in code but carry old TTM "Test Case" keys, while the Confluence page uses the migrated Xray keys — so they can't line up by key. They're correctly counted as automated; the real fix is the TTM→Xray @TmsLink remap (separate work), not adding them to the page under old keys. The registry just surfaced the drift automatically.

The /automation-coverage command (#1964)

Since the bot keeps CI green, the command is for what it doesn't do: pushing the Confluence page and curating the backlog.

/automation-coverage            # curate regression-suite.csv, regenerate, sync the page
python3 scripts/automation_coverage_report.py --check          # CI gate
python3 scripts/automation_coverage_report.py --confluence-html # page body for the manual sync
